# 잡다 용어

## 프로그래밍에서의 Mutation (Mutation in Programming)

진화 알고리즘 외에도, **"Mutation"**이라는 용어는 프로그래밍에서 다음과 같은 의미로도 사용될 수 있습니다.

- **상태 변경 (State Mutation):**
    - *함수형 프로그래밍(Functional Programming)**과 대비되는 개념으로, **변수나 객체의 값(상태)을 변경(mutate)**하는 연산을 의미합니다.
    - 예를 들어, 객체 지향 언어에서 `a = a + 1`이나 배열의 특정 인덱스 값을 바꾸는 작업은 상태를 변화시키는(mutating) 연산입니다. **`set!`** (Scheme 언어)처럼 상태 변경을 나타내는 명명 규칙을 따르기도 합니다.
- **돌연변이 테스트 (Mutation Testing):**
    - 소프트웨어 공학에서 **테스트 스위트(test suite)의 품질을 평가**하는 기법입니다.
    - 원래 프로그램 코드에 의도적으로 작은 오류(돌연변이)를 주입하고, 기존 테스트 케이스가 이 **돌연변이를 발견하고 실패(kill)**하는지 확인합니다. 만약 테스트가 돌연변이를 발견하지 못하면, 테스트 스위트의 품질이 낮다고 판단합니다.
