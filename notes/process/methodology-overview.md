# 개발방법론

- **그린필드(Greenfield)**
    
    완전히 “빈 땅”에서 새로 만드는 프로젝트. 기존 제약(레거시, 데이터 모델, 배포 파이프라인 등)이 거의 없음.
    
    - 장점: 최신 스택·아키텍처 자유롭게 선택, 기술 부채 초기 최소.
    - 단점: 요구사항/범위 불확실성, 0→1 인프라·보안·운영까지 전부 세팅 필요, 일정 리스크 큼.
    - 예: 신규 서비스 런칭, 내부 도구를 처음부터 설계·개발.
- **브라운필드(Brownfield)**
    
    이미 존재하는 시스템/코드/인프라 위에서 개선·확장하거나 교체(모던화)하는 프로젝트.
    
    - 장점: 도메인·사용자·데이터가 이미 있고, 가설 검증이 빠름. 배포/운영 체계 활용 가능.
    - 단점: 레거시 제약(의존성, 릴리즈 프로세스, DB 스키마), 기술 부채 상속, 변경 영향 범위가 큼.
    - 예: 모놀리스를 마이크로서비스로 분해, Vue2→Vue3 마이그레이션, 기존 ERP 연동 고도화.

### 언제 어떤 걸 택해?

- **그린필드**: 기존이 비즈니스 목표를 근본적으로 못 받쳐주거나 완전 다른 시장/제품을 시도할 때.
- **브라운필드**: 사용자/데이터/운영 연속성이 중요하고 다운타임을 최소화해야 할 때.
