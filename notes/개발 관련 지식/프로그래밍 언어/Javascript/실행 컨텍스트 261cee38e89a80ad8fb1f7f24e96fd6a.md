# 실행 컨텍스트

# 실행 컨텍스트란?

실행 컨텍스트는 자바스크립트 코드가 실행되는 환경을 의미합니다. 자바스크립트 엔진이 코드를 평가는 과정에서 실행되며, 실행시에 참고하는 정보입니다. 

### 실행 컨텍스트 종류 
- Global(전역, 함수 외부의 코드), Functional, Eval

실행 컨테스트가 생성되고 변수의 scope를 설정할 때 호이스팅이 일어나게 됩니다. 

실행시에는 렉시컬 환경을 바탕으로 코드가 순차적으로 실행됩니다. 

각각의 실행 컨텍스트는 물리적으로 객체의 형태를 띄며, 변수 환경(VO), 렉시컬 환경, this 바인딩 정보를 갖고 있어서, 어떤 변수를 쓸 수 있는지, 스코프 체인이 어떻게 연결되는지를 관리합니다

전역 실행 컨텍스트는 프로그램 시작 시 하나만 생성되고, 함수가 호출될 때마다 새로운 함수 실행 컨텍스트가 만들어져 콜 스택에 쌓인 순서대로 처리됩니다.

## 구성요소

- 렉시컬 환경

## 렉시컬 환경이란?

1. **환경 레코드 (Environment Record)**
    - 현재 스코프의 변수와 함수 선언을 저장하는 공간
    - `var`, `let`, `const`, 함수 선언 등이 여기에 기록됨
2. **외부 렉시컬 환경 참조 (Outer Lexical Environment Reference)**
    - “상위 스코프”를 가리키는 포인터
    - 이걸 따라가면 스코프 체인을 통해 상위 변수들을 찾을 수 있음
    - 렉시컬 환경은 실행 컨텍스트 안에서 변수를 저장하고, 외부 스코프와 연결하는, 실행 컨텍스트를 구성하는 컴포넌트 입니다.
    
    변수는 환경 레코드에 저장되고, 스코프 체인은 외부 렉시컬 환경 참조를 통해 구현됩니다.
    
    클로저가 동작하는 원리도 결국 함수가 외부 렉시컬 환경을 기억하기 때문입니다.
    
    **렉시컬 환경은 “변수, 함수 선언, 외부 스코프 참조 정보”를 담는 객체**예요.
    
    실행 컨텍스트가 생성될 때 같이 만들어지고,
    
    어떤 변수를 참조할 수 있는지, 스코프 체인이 어떻게 연결되는지를 관리합니다.
    

# 스코프

모든 식별자는 자신이 선언된 위치에 의해 다른 코드가 식별자 자신을 참조할 수 있는 유효 범위가 결정된다. 이를 스코프라 한다. 즉, 스코프는 식별자가 유효한 범위를 말한다. 
자바스크립트 엔진이 식별자를 검색할 때 사용하는 규칙이다.

## 렉시컬 스코프

정적 스코프라고도 하며, 함수가 호출되는 시점에 동적으로 상위 스코프를 결정하는 것을 렉시컬 스코프라고 한다. 

## 함수 레벨 스코프

**변수의 유효 범위(scope)가 함수 단위로 결정된다**는 개념이에요.

- `var`: **함수 레벨 스코프** → 오직 함수 단위로만 유효 범위가 생김.
- `let`, `const`: **블록 레벨 스코프** → `{}` 블록마다 스코프가 생김.
- 이 때문에 현대 자바스크립트에서는 대부분 `let`과 `const`를 권장합니다.
